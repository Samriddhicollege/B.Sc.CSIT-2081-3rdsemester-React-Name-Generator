# Project Report: GeniusNames Generator
**Date:** March 24, 2026

## 1. Project Overview
GeniusNames is a minimalist, high-performance React application designed to synthesize realistic names for projects, characters, and professional use. The project is exclusively text-focused to provide maximum clarity and speed.

---

## 2. Technical Architecture
The project follows the **Parent -> Container -> UI Components** design pattern to ensure scalability and clean code separation.

- **App (Parent)**: The root entry point managing the global layout and theme.
- **NameGenerator (Container)**: The internal engine handling state, API synthesis, and library synchronization.
- **UI Components (Reusable)**: Reusable `Button` and `Card` elements that handle interaction logic.

---

## 3. Core Functions: What, How, and Why

### A. `fetchNames()`
- **What**: Retrieves a batch of names based on specific filters.
- **How**: Performs an asynchronous `fetch` request to the **`randomuser.me`** API. For localized needs, it utilizes a custom **Nepal** dataset.
- **Why**: To provide consistent, real-world data without the need for manual data entry.

### B. `useLocalStorage` (Custom Hook)
- **What**: Manages state persistence across browser sessions.
- **How**: Uses a `useEffect` side effect to serialize and store data in `window.localStorage`.
- **Why**: **Reliability.** This ensures that user-saved names are never lost upon page refresh.

### C. CRUD Matrix (Complete Lifecycle)
- **Create**: Saving generated names into the local "Library."
- **Read**: Dynamic list rendering via `.map()` with unique UUID keys.
- **Update**: Allows users to add a custom "Tag" or "Nickname" to any saved name.
- **Delete**: Supports removing individual entries or resetting the entire library.
